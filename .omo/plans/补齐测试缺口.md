# 补齐测试缺口

> 基于 2026-07-15 全量测试覆盖分析，按优先级分 6 批执行。

## Batch 2 — Provider 网络错误处理测试（P0）

**目标**：覆盖 Provider 客户端对网络故障、超时、非 JSON 响应、rate limit 的容错。
**文件**：新建 `tests/test_provider_errors.py`

### 参考模式
需要 mock HTTP 请求（`urllib`/`httpx`）。现有 `ScriptedModelClient` 模式见 `nova/testing.py`。

### 测试清单

| # | 测试 | 验证点 |
|---|------|--------|
| 2.1 | `test_connection_timeout_raises_model_error` | 网络超时不崩，返回带 timeout 信息的 ModelError |
| 2.2 | `test_invalid_json_response` | API 返回非 JSON → 解析错误+重试 |  
| 2.3 | `test_http_503_retry` | 503 → 重试（最多 N 次） |
| 2.4 | `test_http_429_rate_limit` | 429 + Retry-After header → 等待后重试 |
| 2.5 | `test_empty_response_body` | 空响应体 → 友好错误 |
| 2.6 | `test_max_retries_exceeded` | 连续失败超过上限 → 最终错误，不无限重试 |
| 2.7 | `test_invalid_api_key_returns_actionable_error` | 401 → 错误信息提示 key 无效 |

**实现提示**：`nova/providers/` 下有 `base.py`（抽象基类）和 `clients.py`（具体客户端如 `AnthropicCompatibleModelClient`）。测试用 `pytest` 的 `monkeypatch` 或 `unittest.mock.patch` 拦截 `http.client.HTTPSConnection` / `urllib.request.urlopen` 来控制响应。

### 验证
```bash
python -m pytest tests/test_provider_errors.py -v
```
